Users API

The Users API allows you to create, retrieve, update, and delete users in your application.

GET /api/users
GET

Returns a list of users. The users are returned sorted by creation date, with the most recently created users appearing first.

Query Parameters
Parameter Type Description
limit integer optional A limit on the number of objects to be returned. Default is 10.
offset integer optional Number of items to skip before starting to collect the result set.
role string optional Filter users by role (e.g. 'admin', 'customer').
Example Request
fetch('https://api.example.com/api/users?limit=5&role=admin') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));
Example Response
{ "data": [ { "id": "usr_1234567890", "name": "John Doe", "email": "john.doe@example.com", "role": "admin", "created_at": "2025-03-01T12:00:00Z" }, { "id": "usr_0987654321", "name": "Jane Smith", "email": "jane.smith@example.com", "role": "admin", "created_at": "2025-02-15T09:30:00Z" } ], "meta": { "total": 42, "limit": 5, "offset": 0 } }
POST /api/users
POST

Creates a new user in the system.

Request Parameters
Parameter Type Description
name string required The full name of the user.
email string required The email address of the user. Must be unique.
password string required The user's password. Must be at least 8 characters long.
role string optional The user's role. Defaults to 'customer'.
Example Request
fetch('https://api.example.com/api/users', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer YOUR_API_KEY' }, body: JSON.stringify({ name: 'Alex Johnson', email: 'alex.johnson@example.com', password: 'securepassword123', role: 'manager' }) }) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));
Example Response
{ "id": "usr_2468101214", "name": "Alex Johnson", "email": "alex.johnson@example.com", "role": "manager", "created_at": "2025-03-10T10:15:30Z" }
GET /api/users/:id
GET

Retrieves a user by their unique identifier.

Path Parameters
Parameter Type Description
id string required The unique identifier of the user to retrieve.
Example Request
fetch('https://api.example.com/api/users/usr_1234567890', { headers: { 'Authorization': 'Bearer YOUR_API_KEY' } }) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));
Example Response
{ "id": "usr_1234567890", "name": "John Doe", "email": "john.doe@example.com", "role": "admin", "created_at": "2025-03-01T12:00:00Z", "last_login": "2025-03-10T08:45:22Z", "account_status": "active" }
PUT /api/users/:id
PUT

Updates an existing user's information.

Path Parameters
Parameter Type Description
id string required The unique identifier of the user to update.
Request Parameters
Parameter Type Description
name string optional The updated full name of the user.
email string optional The updated email address of the user. Must be unique.
role string optional The updated role of the user.
account_status string optional The updated account status (e.g. 'active', 'suspended').
Example Request
fetch('https://api.example.com/api/users/usr_1234567890', { method: 'PUT', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer YOUR_API_KEY' }, body: JSON.stringify({ name: 'John M. Doe', role: 'super_admin' }) }) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));
Example Response
{ "id": "usr_1234567890", "name": "John M. Doe", "email": "john.doe@example.com", "role": "super_admin", "created_at": "2025-03-01T12:00:00Z", "updated_at": "2025-03-10T11:22:33Z", "account_status": "active" }
DELETE /api/users/:id
DELETE

Permanently deletes a user. This action cannot be undone.

Path Parameters
Parameter Type Description
id string required The unique identifier of the user to delete.
Example Request
fetch('https://api.example.com/api/users/usr_1234567890', { method: 'DELETE', headers: { 'Authorization': 'Bearer YOUR_API_KEY' } }) .then(response => { if (response.status === 204) { console.log('User successfully deleted'); } }) .catch(error => console.error('Error:', error));
Response

Returns HTTP 204 No Content on success with an empty response body.
